Theme scoping with nested components

Theme styles are not imported in component's stylesUrls property of @Component decorator but in the main styles.scss file. Because of this, theme styles are NOT scoped to the component automatically.

We have to use specific selectors to prevent styles from leaking into child components. This can be achieved by using > (child selectors) in css rules to enhance their specificity. For example checkout theme file of this component:
